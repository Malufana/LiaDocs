    <TabItem value="arquitetura" label="Arquitetura Geral">
        ## Visão Geral
        O projeto foi estruturado com base em uma arquitetura de camadas, separando responsabilidades entre o front-end, back-end e banco de dados. Essa abordagem visa garantir maior escalabilidade, manutenibilidade e organização do sistema.

        - **Front-End**: Desenvolvido em Vue.js, responsável pela interface do usuário;
        - **Back-End**: API desenvolvida com Python e FastAPI, que gerencia a comunicação com o front-end e a OpenAI da Azure;
        - **Banco**: Utiliza o próprio banco da Azure, o Blob Storage, onde estão armazenados os contratos.

        ## Componentes da Arquitetura
        ### Front-End
        - Framework: **Vue.js**
        - Funcionalidades: consome a API, renderiza dados e gerencia rotas;
        - Comunicação: requisições HTTP usando o `axios`.

        ### Back-End
        - Framework: **FastAPI**
        - Função: processa dados, gera resumos com a IA e retorna resultados para o front-end;
        - Integração com IA: usa OpenAI/Azure para a geração de resumos.

        ### Banco de dados
        - Tipo: **Azure Blob Storage**
        - Utilização: armazena os contratos e seus templates.

        ## Fluxo de dados
        1. O usuário acessa o front-end e entra em um contrato dentro da aba "Criados Recentemente".
        2. O front-end envia o base64 do contrato para a API.
        3. A API analisa se o base64 é um HTML ou um PDF, extrai o texto, processa com a IA e retorna o resumo.
        4. O front-end exibe o resumo para o usuário.

    <TabItem value="publico" label="Público-Alvo e Benefícios">
        ## Público-Alvo
        Este projeto é voltado para:
        - **Profissionais jurídicos** que lidam com grandes volumes de contratos;
        - **Empresas e departamentos legais** que precisam automatizar e agilizar o processo de leitura e análise contratual;
        - **Usuários sem formação jurídica** que precisam entender contratos complexos de forma simples.

        ## Problemas enfrentados
        - A leitura de contratos longos é **demorada e repetitiva**;
        - Termos jurídicos dificultam a **compreensão por pessoas não especializadas**;
        - Falta de ferramentas que resumam automaticamente cláusulas e termos relevantes.

        ## Como o projeto resolve
        A solução desenvolvida utiliza inteligência artificial para:
        - Extrair automaticamente o texto de contratos em PDF;
        - Resumir o conteúdo de forma clara e objetiva;
        - Disponibilizar os resumos em uma interface visual e acessível.

        ## Benefícios
        - **Economia de tempo**: análise contratual em segundos;
        - **Compreensão facilitada**: resumos simplificam termos técnicos;
        - **Aumento da produtividade**: usuários podem focar em decisões, não na leitura;
        - **Uso corporativo**: ideal para quem lida com grande volume de documentos legais.
